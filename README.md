# solid-express

Este projeto explora a aplicação dos princípios SOLID no desenvolvimento orientado a objetos (OOP), com foco em Java. 

## Princípios SOLID

1. **Single Responsibility Principle (SRP):**  
    Cada módulo ou classe deve ter uma única responsabilidade, facilitando manutenção e testes.

2. **Open/Closed Principle (OCP):**  
    Os componentes devem ser abertos para extensão, mas fechados para modificação, permitindo adicionar novas funcionalidades sem alterar o código existente.

3. **Liskov Substitution Principle (LSP):**  
    Subclasses devem poder substituir as classes base sem afetar o funcionamento do sistema.

4. **Interface Segregation Principle (ISP):**  
    Prefira interfaces específicas e pequenas ao invés de grandes e genéricas, tornando o código mais flexível.

5. **Dependency Inversion Principle (DIP):**  
    Dependa de abstrações e não de implementações concretas, promovendo desacoplamento e testabilidade.
